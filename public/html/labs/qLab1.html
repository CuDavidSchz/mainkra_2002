Preguntas Laboratorio 1 - HTML

  1. P: ¿Cuál es la diferencia entre Internet y la World Wide Web (WWW)?

    R: Internet es una red de computadoras conectadas y WWW es una colección de páginas sobre esas computadoras

  2. P: ¿Cuáles son las partes de un URL?

    R: Se compone de los siguientes elementos:

    1. Protocolo de comunicación
    2. Dominio
    3. Ruta (carpeta y archivo)
    4. Parámetros

  3. P: ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    R: Sus propósitos son:

    1. GET: Solicita datos al servidor.

    2. HEAD: Verifica si un recurso existe. Devuelve solo encabezados.

    3. POST: Envía datos al servidor para crear un nuevo recurso.

    4. PUT: Modifica o reemplaza un recurso existente.

    5. PATCH: Modifica parcialmente un recurso existente.

    6. DELETE: Elimina un recurso.

  4. P: ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    A: POST los envía en el cuerpo de la solicitud, lo que los hace menos visibles y más seguros.

  5. P: ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    A: GET solicita datos del servidor pero no realiza cambios en él.

  6. P: Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    A: Indica que se recibió, procesó y se realizó correctamente una solicitud.

  7. P: ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    A: El error 404 indica que el recurso no fue encontrado. Sin embargo el error debe repararse siempre y cuando la causa del mismo no sea que el usuario ha escrito mal la URL.

    Esto porque todas las demás causas indican que hay un error del lado del servidor.

  8. P: ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    A: Este código dice que existe un problema interno en el servidor, por lo que si es responsabilidad del desarrollador repararlo.

  9. P: ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    A: El término depreciado o desaprobado quiere decir que esa etiqueta no es recomendada para para usarse ya que existen mejores opciones para lograr el mismo resultado.

    La mayoría de los atributos depreciados tienen que ver con el estilo.

    Algunos ejemplos son:

    • font: sustituido por los atributos font en CSS.

    • center: sustituido por text-align en CSS.

    • big y small: se recomienda modificar tamaños en CSS.

  10. P: ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    A: Las principales diferencias son:

    • Semántica: antes se usaba div y span y ahora existen etiquetas especializadas para dividir el contenido.

    • Soporte multimedia: antes se necesitaba de flash para reproducir vídeos e imágenes.

    • Inputs de formularios: ahora existen tipos de datos especiales en los formularios.

  11. P: ¿Qué componentes de estructura y estilo tiene una tabla?

    A: La esructura que sigue una tabla en HTML es:

    • table: es el cuerpo principal de la tabla.

    • tr: define una fila en la tabla.

    • th: define una columna en la tabla.

    • td: define una celda de datos.
